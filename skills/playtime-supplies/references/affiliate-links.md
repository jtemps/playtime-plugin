# Affiliate links

Playtime can include Amazon affiliate links in shopping lists. We use **search URLs** (not specific product/ASIN URLs) so links don't rot and the parent picks the actual product.

## Default affiliate tag

The plugin ships with a default Amazon Associates tag. Users can override it with their own tag in their profile.

```
DEFAULT_AFFILIATE_TAG = "dailyplay-20"
```

> Plugin default is `dailyplay-20`. Forks should swap in their own Amazon Associates tracking ID, or set `household.affiliateTag` to `""` in their profile to opt out entirely.

### Resolution order

1. If `household.affiliateTag` is set in `~/.playtime/profile.json`, use that.
2. If `household.affiliateTag === ""` (empty string), generate plain Amazon search URLs WITHOUT a tag — this is how a downstream user opts out.
3. Otherwise, fall back to `DEFAULT_AFFILIATE_TAG` above.

## Building a search URL

Format:

```
https://www.amazon.com/s?k=<URL_ENCODED_KEYWORDS>&tag=<AFFILIATE_TAG>
```

Examples:

| Keyword | URL |
|---|---|
| `kids safety scissors blunt tip` | `https://www.amazon.com/s?k=kids+safety+scissors+blunt+tip&tag=dailyplay-20` |
| `washable tempera paint kids` | `https://www.amazon.com/s?k=washable+tempera+paint+kids&tag=dailyplay-20` |
| `play doh classic 10 pack` | `https://www.amazon.com/s?k=play+doh+classic+10+pack&tag=dailyplay-20` |

URL-encode spaces as `+` (Amazon's search query format).

## When to use no tag

If `household.affiliateTag` is explicitly set to `""` (empty string), generate plain search URLs:

```
https://www.amazon.com/s?k=kids+safety+scissors+blunt+tip
```

This is the user's opt-out path — no tag, no affiliate revenue.

## FTC disclosure (REQUIRED)

Every shopping list / packet that includes affiliate links MUST include a clear disclosure. Use one of these (depending on tone):

### Long form (preferred for first-time / standalone shopping lists)

> **Note about links**: Amazon links on this page are affiliate links. If you buy through them, the plugin's owner may earn a small commission at no extra cost to you. You can replace the affiliate tag with your own (or none) by running `/edit-playtime-household` and updating "Amazon affiliate tag".

### Short form (for footers in regular packets)

> 🔗 *Amazon links are affiliate links. Run `/edit-playtime-household` to use your own tag (or none).*

If `household.affiliateTag === ""`, you can OMIT the disclosure entirely — there's nothing to disclose.

## When to include affiliate links

- ✅ In shopping lists generated by `/playtime-shop`
- ✅ In `/playtime-checkup` reports when "missing supplies" is flagged
- ✅ In an activity packet's "for grown-ups" footer **only when** a needed material is genuinely missing AND the user hasn't already been pitched it recently — don't pitch the same item every day
- ❌ Never in kid-facing copy
- ❌ Never on every packet — that turns Playtime into a shopping channel

## Implementation pseudocode

```javascript
function affiliateUrl(searchKeyword, profile) {
  const encoded = searchKeyword.trim().split(/\s+/).join("+");
  const tag = profile.household.affiliateTag !== undefined
    ? profile.household.affiliateTag      // user-set (might be "")
    : "dailyplay-20";               // plugin default

  return tag
    ? `https://www.amazon.com/s?k=${encoded}&tag=${tag}`
    : `https://www.amazon.com/s?k=${encoded}`;
}
```

(The actual implementation is markdown-link rendering inside generated content — but the URL construction follows this logic.)

## Things to verify before publishing

- The default tag string is YOUR real Amazon Associates ID
- Disclosure text appears wherever affiliate links appear
- Plugin README has a section explaining default tag + how to override
