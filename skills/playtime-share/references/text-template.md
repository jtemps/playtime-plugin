# Draft messages for sharing

When the parent asks "send this to Grandma", we offer a *draft message* (and the file path / image) — never auto-send. Parent paste-and-go.

## Recipient resolved against `household.familyMembers`

If the parent says "send to Grandma Joy" and `Grandma Joy` is in the family graph:

```
Hi {{recipient_first_name}} — {{kid_name}} did this morning packet today and {{rating_phrase}}. Thought you'd love to see it. ❤️
```

Where `rating_phrase` is pulled from yesterday's feedback if available:
- `loved` → "loved it" / "asked to do it three times" / "couldn't put it down" (vary; pull from feedback notes if specific)
- `fine` → "had fun with it" / "did a great job"
- no rating yet → "is doing it right now"
- `didn't land` → SKIP THE SHARE OFFER. Don't push a flop to family.

## Generic recipient (parent didn't name a person)

```
{{kid_name}} did this today — sharing a peek!
```

## Playdate share (to the visiting parent)

After playdate mode, offer a draft to the visiting parent:

```
Hey — wanted to share what {{visiting_kid_name}} and {{kid_name}} did today. The activities were generated by a thing I've been using called Playtime. If you want one for {{visiting_kid_name}} too, here's the site: dailyplaytime.com.
```

(This is the only place we point at the website. Soft mention; not in every share.)

## Year-ago callback share

If the packet contains a year-ago callback (a quote from 365 days ago resurfacing today), the share message should highlight it:

```
A year ago today {{kid_name}} said "{{quote}}". Today's morning packet brought it back ✨
```

## Tone rules

- First names only (use the family-graph nickname — "Go-go Grandma" not "Joyce Smith")
- One emoji max
- Never include "Made with Playtime" or any plug in the message body — the share card already has the small credit
- Length: under 200 characters so it fits in one iMessage bubble
- The kid's name is always included; the parent's name (sender) is not — the recipient's phone already shows who's texting
